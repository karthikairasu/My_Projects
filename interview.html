Call, Apply, and Bind
  • Call
  • With the call() method, you can write a method that can be used on different objects. 
    •                 const person = {
                  fullName: function(city, country) {
                    return this.firstName + " " + this.lastName + "," + city + "," + country;
                  }
                }

                const person1 = {
                  firstName:"John",
                  lastName: "Doe"
                }; 

                var personres = person.fullName.call(person1, "Oslo", "Norway");
                console.log(personres);  
              
                  function Company(name, age){
                this.name = name;
                this.age = age;
              }

              function Student1(name, age){
                Company.call(this, name, age);
                this.sec = "A";
              }

              function Student2(name, age){
                Company.call(this, name, age); 
                this.sec = "B";
              }

              var res1 = new Student1("siva", 25);
              console.log(res1.name, res1.age, res1.sec);  

              var res1 = new Student2("logi", 28);
              console.log(res1.name, res1.age, res1.sec);

1. What is Hoisting ?

          
Hoisting is JavaScript's default behavior of moving all declarations to the top of the current scope (to the top of the current script or the current function).
            

              Example:
              
                x = 5;
                console.log(x);
                var x;
              
              * var declarations are hoisted and initialized with undefined.
              * let and const declarations are hoisted but not initialized (they exist in the temporal dead zone).
              

            

          

        

        

          
2. What is Closure ?

          
A closure is the combination of a function bundled together (enclosed) with references to its surrounding state (the lexical environment). In other words, a closure gives you access to an outer function's scope from an inner function.
            

              Example:
              
                function buildGreeting() {
                  let message = "Hello";
                  function greetUser() {
                    console.log(message);
                  }
                  return greetUser;
                }
                let hello = buildGreeting();
                hello();

              * https://blog.hubspot.com/website/javascript-closure
              

            

            IIFE and Closures:

                // This helps avoid the closure issue with setTimeout.
                (function (id) {
                  setTimeout(function () {
                    console.log('seconds: ' + id);
                  }, id * 1000);
                })(id);
              }
            

          

        

        

          
3. Higher Order Function ?

          
A higher-order function is a function that either takes one or more functions as arguments or returns a function as its result.
            

              Example:
              
                // A higher-order function that takes a function as an argument
                function applyOperation(arr, operation) {
                    return arr.map(operation);
                }

                // A simple function to double a number
                function double(x) {
                    return x * 2;
                }

                // Using the higher-order function
                const numbers = [1, 2, 3, 4];
                const doubled = applyOperation(numbers, double);
                console.log(doubled); 
              

            

          

        

        

          
4. Javascript currying ?

          
Currying is a functional programming technique in JavaScript (and other languages) where a function is transformed into a sequence of functions, each taking a single argument. Currying helps you avoid passing the same variable multiple times, and it helps you create a higher order function.
            

              Example:
              
                // A curried function for adding three numbers
                function add(x) {
                    return function(y) {
                        return function(z) {
                            return x + y + z;
                        };
                    };
                }

                // Using the curried function
                const addFive = add(5);          // Fix the first argument
                const addFiveAndThree = addFive(3); // Fix the second argument
                const result = addFiveAndThree(2); // Now we fix the last argument

                console.log(result); // Output: 10 (5 + 3 + 2)
              

            

          

        

        

          
5. What are object prototypes ?

          
Every object in JavaScript has a built-in property, which is called its prototype. which is a reference to another object. This is used to establish a chain of objects from which the object can inherit properties and methods. This chain is known as the prototype chain.
            

              Example:
              
                function Person(name) {
                  this.name = name;
                }
                
                Person.prototype.greet = function() {
                  console.log(`Hello, my name is ${this.name}`);
                };
                
                const person1 = new Person("Alice");
                person1.greet();

11. Javascript Throttling and Debouncing ?

          
Throttling and debouncing are techniques used to control the rate at which a function is invoked, particularly in scenarios where frequent events (like user interactions) can lead to excessive calls and potentially impact performance. 
            Throttling - Throttling limits the number of times a function is executed over a certain time period. It ensures that the function is called at a controlled, steady rate, regardless of how frequently the event triggering the function occurs.
            Debouncing - Debouncing is a technique that delays the execution of a function until a certain amount of time has passed since the last event's occurrence. It's useful when you want the function to be called after a period of inactivity, and it prevents rapid consecutive calls.

12. Event Bubbling and Event Capturing?

          
Event bubbling is the default event propagation behavior. In event bubbling, when an event occurs on an element, it first triggers the event handler on the target element (the element the event was originally triggered on), then it propagates up the DOM tree to the parent elements, and so on, until it reaches the root of the document.
            Event capturing is the opposite of event bubbling. In event capturing, the event starts from the root of the document (or the highest level of the DOM) and then propagates downward through the DOM tree, reaching the target element last.
